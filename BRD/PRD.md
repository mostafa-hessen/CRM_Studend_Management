## 🏛️ 1. الرؤية والهوية

**نظام إدارة الطلاب** هو منصة إدارية متكاملة ومتخصصة، بُنيت من الصفر لتلبية احتياجات **الشيخ وائل** وفريقه في متابعة الطلاب وإدارة الحملات التعليمية بكفاءة عالية. يتجاوز النظام حدود الفكرة الأولية ليكون منتجاً برمجياً ناضجاً، مبنياً على **معمارية معيارية (Modular Architecture)**، مزوداً بـ**نظام توجيه (SPA Router)**، و**غطاء اختبارات وحدة (Unit Tests)** حقيقي.

لم يُبنَ هذا النظام ليكون مجرد نموذج أولي — بل هو **نظام جاهز للإنتاج** يعمل بالكامل من المتصفح دون الحاجة لأي خادم خلفي، بالاعتماد على **LocalStorage** كقاعدة بيانات محلية متينة.

## 👥 3. المستخدمون والصلاحيات (RBAC)

يعتمد النظام على نموذج **Role-Based Access Control** مدمج بالكامل في المتصفح.

| الدور | المستخدم | الصلاحيات |
|---|---|---|
| 👑 **مدير النظام** | الشيخ وائل | وصول كامل: إدارة المستخدمين، حذف البيانات، عرض سجل النشاطات، التقارير الكاملة |
| 👩‍💼 **موظف** | سارة / حبيبة | إضافة وتعديل الطلاب، إدارة الحملات، المتابعات — **بدون** صلاحية الحذف أو إدارة المستخدمين |

## ⚙️ 4. الميزات الوظيفية (Functional Features)

### 4.1 — لوحة التحكم (Dashboard)
نقطة انطلاق كل جلسة عمل. تعرض في لمحة واحدة:
- **بطاقات إحصائية ذكية:** إجمالي الطلاب، المسجلون، المهتمون، المتصل بهم، لم يردوا.
- **قائمة المتابعة اليومية:** تُكتشف تلقائياً وتعرض الطلاب الذين يستحقون اتصالاً في تاريخ اليوم.
- **يُحدَّث فورياً** عند أي تغيير في البيانات.

---

### 4.2 — إدارة الطلاب (Students Management)
القلب النابض للنظام:

- **إضافة طالب:** نموذج ذكي يشمل (الاسم، هاتف الطالب، هاتف ولي الأمر، الصف الدراسي، المدرسة، ملاحظات إضافية).
- **تعديل شامل:** تحديث جميع بيانات الطالب بما فيها أرقام الهواتف المتعددة.
- **حذف آمن:** مع تأكيد قبل الحذف وتسجيل العملية في سجل النشاط.
- **بحث وتصفية متقدم:** فلترة فورية بالاسم أو الهاتف أو الصف الدراسي.
- **قاعدة تحقق صارمة (Validation):** لا يُسمح بتكرار رقم هاتف الطالب مطلقاً.
- **حالات التواصل (Communication Status):**

| الرمز | الحالة |
|---|---|
| ✅ | مهتم |
| ❓ | متردد |
| ❌ | غير مهتم |
| ⏳ | اتصل لاحقاً |
| 🔇 | لم يرد |
| 🌟 | تم التسجيل |

---

### 4.3 — إدارة الحملات (Campaigns)
- **إنشاء حملات تعليمية** بأسماء وتواريخ وأهداف مخصصة.
- **إضافة الطلاب للحملة** بشكل مرن مع تجنب التكرار.
- **تتبع التقدم:** شريط تقدم بصري يعرض (عدد المسجلين من مجموع طلاب الحملة) كنسبة مئوية فورية.
- **إحصائيات الحملة:** معدل التحويل الفعلي لكل حملة على حدة.

---

### 4.4 — إدارة الصفوف الدراسية (Classes)
- إنشاء وتعديل وحذف الصفوف الدراسية.
- ربط الطلاب بصفوفهم تلقائياً.
- عرض إحصائيات تفصيلية لكل صف.
- مودول مستقل قابل للاختبار الآلي ✅.

---

### 4.5 — نظام المتابعات (Follow-ups)
- تحديد تاريخ متابعة دقيق لكل طالب يطلب الاتصال لاحقاً.
- التنبيه التلقائي في لوحة التحكم عند حلول الموعد.
- إمكانية تحديث حالة المتابعة بعد إجراء الاتصال.

---

### 4.6 — التقارير والإحصائيات (Analytics)
- **رسم بياني** لتوزيع حالات التواصل.
- **نسبة التحويل** (Conversion Rate) من اهتمام الى تسجيل.
- **أداء الفريق:** إحصائيات مقارنة بين الموظفين.

---

### 4.7 — إدارة المستخدمين (Users — Admin Only)
- إضافة حسابات الموظفين وحذفها.
- تغيير كلمات المرور بأمان.
- مخزّنة بشكل مُشفَّر في LocalStorage.

---

### 4.8 — سجل النشاطات (Audit Log — Admin Only)
- تسجيل كل إجراء يتم في النظام (إضافة، تعديل، حذف) بـ:
  - اسم المستخدم المنفِّذ.
  - نوع العملية.
  - التاريخ والوقت الدقيقين.
- يمنح المدير رؤية كاملة وشفافة على عمل الفريق.

---

### 4.9 — النسخ الاحتياطية (Backup)
- تصدير كامل بيانات النظام بصيغة **JSON**.
- استيراد نسخة احتياطية لاستعادة البيانات بسرعة.

## 🏗️ 5. المعمارية التقنية (Technical Architecture)

هذا ليس تطبيقاً أحادي الملف (monolithic) — بل نظام مُهندَس بمعمارية **SPA معيارية** صُممت للقابلية للتوسع والاختبار.

```
assets/js/
├── app.js                  # نقطة الدخول الرئيسية (Entry Point)
├── core/
│   ├── router.js           # نظام التوجيه بين الصفحات (SPA Router)
│   └── state.js            # إدارة الحالة المركزية (Centralized State)
└── modules/
    ├── auth.js             # المصادقة وإدارة الجلسة
    ├── students.js         # منطق الطلاب (CRUD كامل)
    ├── campaigns.js        # منطق الحملات
    ├── classes.js          # منطق الصفوف الدراسية
    ├── users.js            # إدارة المستخدمين (Admin)
    ├── dashboard.js        # مجمِّع لوحة التحكم
    ├── storage.js          # طبقة التخزين المحلي
    ├── backup.js           # تصدير/استيراد البيانات
    └── ui.js               # مساعدات واجهة المستخدم
```

### مبادئ المعمارية المُطبَّقة:
- **Separation of Concerns:** كل وحدة مسؤولة عن نطاق واضح ومستقل.
- **Single Source of Truth:** `state.js` يُدير الحالة المركزية للتطبيق.
- **SPA Router:** التنقل بين الصفحات دون إعادة تحميل (No Page Reload).
- **Modular ES6 Imports:** وحدات مستقلة قابلة للاستيراد والاختبار.

## 🧪 6. جودة الكود والاختبارات

يمتلك النظام **غطاء اختبارات آلي** باستخدام **Vitest**:

| ملف الاختبار | ما يختبره |
|---|---|
| `tests/auth.test.js` | منطق تسجيل الدخول والخروج والتحقق من الأدوار |
| `tests/classes.test.js` | إضافة وتعديل وحذف الصفوف الدراسية |
| `tests/state.test.js` | سلوك إدارة الحالة المركزية |

```bash
# تشغيل الاختبارات
npm test

# تقرير التغطية الكاملة
npm run coverage
```

## 📝 10. الخلاصة

> نظام إدارة الطلاب ليس مجرد تطبيق ويب — إنه **أداة عمل احترافية** صُممت خصيصاً لتيح للشيخ وائل وفريقه التركيز على ما يهم: **الوصول إلى أكبر عدد ممكن من الطلاب، ومتابعتهم حتى التسجيل الفعلي**. يجمع بين **سرعة التطبيقات المحلية** و**قوة الأنظمة المؤسسية** داخل متصفح واحد.
